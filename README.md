# Gravity-Gomoku
项目简介
本项目为计算机专业算法课程大作业，基于 Python + Pygame 实现重力下落式二维五子棋，内置双人对战、AI 人机对战双模式；AI 采用 Minimax 极大极小值算法 + Alpha-Beta 剪枝优化，局面估值区分活棋 / 眠棋，攻防权重平衡，分为简单 / 中等 / 困难三档推演深度；完整实现界面导航、难度选择、先后手切换、悔棋、重置、胜负判定、独立 exe 打包等全套功能。
棋子遵循重力规则：点击任意列，棋子自动下落至该列最底部；横、竖、两条对角线任意方向连成 5 子即获胜。
技术栈

    编程语言：Python 3.8+
    图形库：Pygame 2.5
    AI 核心算法：Minimax 极大极小搜索、Alpha-Beta 剪枝、自定义局面估值函数
    打包工具：PyInstaller

功能清单

    首页：三个功能按钮
        TWO PLAYERS：双人本地对战
        VS AI PLAYER：进入 AI 难度选择页
        HOW TO PLAY：游戏规则说明页
    AI 难度选择页
        三档难度：EASY（推演深度 1）、MEDIUM（深度 2）、HARD（深度 3）
        切换落子先后手：人类先手 / AI 先手
        右上角 BACK 返回首页
    对局棋盘页面
        右上角 BACK HOME 一键返回首页
        左上角实时提示当前操作玩家
        胜利 / 平局文字永久停留，不会自动刷新消失

2. 游戏核心规则

    重力落子机制：点击列，棋子下落至列最低空位
    棋子配色：玩家 1 黑色，玩家 2/AI 白色，深色背景对比度高
    获胜判定：同层横 / 竖 / 斜向连续 5 颗同色棋子获胜；棋盘填满无五子则平局
    操作快捷键
        R：清空棋盘，重新开局
        U：悔棋
            双人模式：仅撤销上一步玩家落子
            人机模式：一次性撤回人类 + AI 两步，回到人类落子前状态

3. AI 智能模块（课程算法核心）

    三层难度梯度，对应不同 Minimax 搜索深度，算力与强度平衡
    Alpha-Beta 剪枝：大幅剔除无意义分支，降低搜索耗时
    强化攻防型局面估值函数（作业重点）
        区分活四、眠四、活三、眠三、活二，不同连线赋予差异化分值
        防守权重高于普通进攻：对手活四、活三优先级高于自身普通连线，AI 主动封堵威胁
        棋盘中心区域额外加分，AI 优先抢占核心点位
    落子逻辑：人类落子后 AI 自动计算最优位置并重力落子

4. 打包部署
支持 PyInstaller 一键打包为独立 Windows exe，无控制台黑窗口，可直接分发运行。

AI 算法设计（课程作业重点阐述）
1. Minimax 极大极小算法

    MAX 层（AI）：选择当前局面估值最高的落子，追求胜利、扩大优势
    MIN 层（人类玩家）：模拟人类最优应对，选取估值最低的走法，预判自身危险
    递归限定搜索深度，区分三档难度，控制 AI 思考耗时

2. Alpha-Beta 剪枝优化
递归过程维护 α、β 边界值，提前截断无搜索价值分支，同深度下搜索效率提升数倍，解决高深度 AI 卡顿问题。
3. 强化防守型局面估值函数（核心改进点）

    对所有 4 个连线方向遍历，统计连续棋子数量
    判断连线两端是否为空，区分活棋（两端无遮挡，威胁极大）与眠棋（一端被堵）
    分值权重规则：
        对手活四 > AI 自身活四，AI 优先封堵必胜威胁
        对手活三 > AI 普通三连，提前阻断成型攻势
        棋盘中心坐标额外加分，引导 AI 抢占关键点位
    同时兼顾进攻与防守，AI 既会主动构建五子连线，也会预判并封锁玩家的潜在胜势。

作业创新点

    重力下落落子机制，区别于传统五子棋点击任意位置落子规则，增加博弈限制
    完善分层页面交互系统，难度、先后手可视化选择
    针对性优化估值函数，大幅提升 AI 防守意识，解决普通贪心 AI 只会进攻不会封堵的缺陷

    可独立打包 exe，无需配置 Python 环境，交付、演示便捷
